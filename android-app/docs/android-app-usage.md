# CS Learning OS Android 使用说明

## 开始使用

安装并打开 `CS Learning OS` 后，应用会在本机初始化内置学习内容。笔记、题卡、复习记录、捕捉条目和备份都保存在设备本地的 Room 数据库中。核心学习流程不依赖账号、桌面端服务或远程后端。

底部导航包含五个入口：`首页`、`捕捉`、`知识库`、`复习` 和 `更多`。

## 首页

![Home 首页](images/android-home.png)

首页汇总本机学习进度，并提供继续学习、新建 Markdown 笔记、搜索和进入待复习内容的快捷入口。这里显示的统计都来自本地数据，不需要联网。

## 捕捉

`捕捉` 用于先保存零散想法、问题、来源线索和临时草稿。新建 Capture Slip 后，你可以继续编辑、归档、恢复或删除它，也可以把它整理成正式 Markdown 笔记。Capture Slip 不会自动写入知识库，最终保存仍由你确认。

## 知识库

![Library 知识库](images/android-library.png)

`知识库` 按 Area 浏览本地笔记。你可以新建 Area，打开节点阅读或编辑，按需搜索，并将暂时不用的节点移到回收站。回收站中的节点可以恢复；永久删除后只能从你之前导出的备份中找回。

创建或编辑笔记时，填写标题、选择 Area，并编辑 Markdown 正文后保存。笔记中的 `:::quiz` 题卡会进入 `复习` 流程。

## 复习

`复习` 显示到期题卡，并支持按 Area 开始学习。先查看题目和答案，再选择 `Again`、`Hard` 或 `Good` 评分；应用会据此更新下一次复习时间。复习队列完全保存在本机，不依赖网络。

## 更多

`更多` 将设置分为四个分区：

- `系统`：语言和外观设置。
- `服务`：可选 AI 服务商配置。
- `数据`：备份、恢复和本地数据工具。
- `指南`：学习流程说明和知识助理入口。

执行恢复、清理本地数据或更换设备前，先从 `更多 -> 数据` 导出备份。恢复会替换当前本地数据；导入文件在写入前会经过格式和大小边界检查，未通过校验的文件不会被导入。

## 复习同步（Study Sync）

`更多 -> 复习同步` 用于让手机与同一局域网内的电脑端协作。它不是完整知识库镜像，也不是备份替代品；电脑端仍负责复杂创建、Markdown/frontmatter、知识图谱和索引维护，手机端只拉取适合随时复习训练的学习子集。

当前手机端同步重点包括：

- 轻阅读节点：用于在手机上快速查看必要正文，而不是承载完整桌面知识图谱。
- 复习卡：用于 `复习` 页面练习。
- 每日一练：电脑端生成并保存 `Daily Bite` 后，手机端可同步到 `更多 -> 每日一练`。
- 随手记、读者问题和复习进度：用于补齐移动端学习闭环。

使用时请在电脑端通过局域网同步入口生成配对令牌，并在手机端填写电脑显示的 Wi-Fi 地址，例如 `http://192.168.1.x:8001`。不要在手机端填写 `127.0.0.1`，因为那只代表手机自己。

如果手机端看不到 Daily Bite，通常不是配对失败，而是电脑端还没有把题目提取并保存为 `bite_card`，或该卡片所在 Area 没有包含在手机端的复习范围中。

## AI 引导

AI 是可选能力，不是应用前提。没有配置 AI 时，笔记、捕捉、搜索、复习、回收站和备份仍可离线使用。

AI 相关的当前边界如下：

- 只接受你手动配置的 `HTTPS` 服务地址。
- API 密钥只保存在本机，并使用 Android Keystore 保护。
- AI 结果始终先作为建议或草稿出现，不会静默写入知识库。
- 任何最终保存都需要你手动确认。

### 配置 AI 服务

1. 打开 `更多 -> 服务`。
2. 填写服务商名称、API Key、Base URL 和模型名称。
3. Base URL 必须是带主机名的 `HTTPS` 地址。
4. 点击 `Save settings` 保存配置。
5. 点击 `Validate` 校验连接；需要时再点击 `Models` 拉取可选模型并完成选择。

如果校验失败，先检查 Base URL、模型名和密钥是否正确，再重试。未通过校验时，不要假设生成或助理对话一定可用。

### 用 AI 草稿整理 Capture Slip

这是从配置到入库的完整路径：

1. 在 `更多 -> 服务` 完成 AI 配置并通过 `Validate`。
2. 进入 `捕捉`，新建或打开一条 Capture Slip。
3. 选择生成 AI 草稿。
4. 先审阅预检信息，确认将发送的 Slip 内容和节点标题上下文。
5. 需要时再次校验服务配置，然后显式点击 `Generate draft`。
6. 生成结果会以可编辑的 Markdown 草稿打开。
7. 逐项检查标题、Area、正文和题卡内容，必要时手动修改。
8. 只有在你确认内容可用后，点击 `Save Markdown`，草稿才会写入知识库。

关闭页面、返回上一级或继续编辑，都不会触发自动保存。AI 草稿只是提案，不是最终记录。

### 使用知识助理

知识助理用于解释概念、整理草稿、辅助复习规划和生成可审阅建议。你可以从首页快捷入口进入，也可以从 `更多 -> 指南 -> Knowledge assistant` 打开助理说明并进入助理界面。

知识助理的工作方式如下：

1. 先确保 `更多 -> 服务` 中的 AI 配置已保存并校验通过。
2. 打开知识助理并输入问题。
3. 应用会先在本地检索相关资料，再附带有限上下文发送给你配置的服务商。
4. 回复会显示引用来源，便于你核对依据。
5. 如果助理提出草稿、修改建议或保存建议，仍需要你逐项审阅并手动确认。

当前实现会将本地检索结果限制在最多 3 条摘录、合计最多 1,200 个字符的上下文范围内。请不要在问题中输入不应发送给外部服务商的敏感信息。

## 数据与隐私

日常学习数据默认只保留在设备本地。只有你主动配置并显式使用 AI 时，相关请求才会发送到你选择的 `HTTPS` 服务商。是否发送某段内容，应以你自己的判断和该服务商的隐私政策为准。

当前版本不提供远程账号、托管云同步、批量 AI 写入、代码阅读器或公式阅读器。可选的 Study Sync 只同步手机复习训练所需的学习子集；需要迁移或恢复完整手机数据时，请使用显式导出和恢复，而不是依赖同步状态。
