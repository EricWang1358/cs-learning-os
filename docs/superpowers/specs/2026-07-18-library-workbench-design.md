# Library Workbench 设计规格

## 目标

将桌面端 Library 从“连续卡片列表”重构为高密度工作台：用户可以按最近更新时间快速扫描知识节点，使用日期分组定位近期内容，并在不离开当前上下文的情况下搜索、筛选、排序和维护学习序号。

本设计只覆盖桌面端 Library（`/nodes`）及其共享搜索工具栏视觉规范。安卓端本阶段不提供序号编辑入口，也不改变现有本地数据语义。

## 页面结构

- 保留左侧导航和右侧节点详情核心功能。
- 主内容区顶部使用紧凑工具栏，不再固定吸顶。
- 工具栏包含搜索、Area/Track 筛选、排序和全部展开/收起操作。
- 节点列表按日期分组；组之间使用轻量分界线和标题，不使用大卡片容器。
- 节点使用高密度横向行：序号、标题、Area/Track、摘要、更新时间和状态。
- 当前选中项使用左侧强调线和文字状态，不使用整块高饱和背景。
- 详情面板保持既有功能，不纳入列表重构范围。

## 时间分组与排序

所有分组和排序以节点 `updated_at` 为主字段，并按北京时间自然日计算：

| 分组 | 范围 |
| --- | --- |
| 今天 | 今天更新 |
| 两天内 | 昨天更新 |
| 一周内 | 前 2 至 7 天更新 |
| 一周前 | 超过 7 天更新 |

分组互斥，空分组隐藏。默认展开“今天”和“两天内”，默认收起旧分组。工具栏提供“全部展开”和“全部收起”；状态仅在当前 Library 会话内保留，离开页面后恢复默认状态。

主排序规则：

1. `updated_at` 降序（最新在前）；
2. 同一时间按 `display_order` 升序；
3. 序号仍相同或缺失时按标题字母序。

搜索、Area/Track 筛选只作用于当前可见结果，分组重新计算；只显示非空分组。

## 搜索工具栏

- 输入即筛选，约 250ms 防抖；按 Enter 时立即执行。
- 搜索词、Area、Track 和排序状态保留在 URL，刷新和返回可恢复。
- 提供清空搜索按钮。
- 工具栏不再全局固定，滚动时不遮挡列表内容。
- Library、Quiz、Question Queue 复用视觉组件，但保留页面各自的搜索语义；Health、Sync、Review 等页面不显示无关搜索控件。

## 序号维护

- 序号按 Area + Track 独立。
- 序号允许间隔，但必须是非空正整数且在同一 Area + Track 内唯一。
- 默认以低对比度文本显示；鼠标悬停高亮。
- 双击序号进入编辑；只有按 Enter 才提交。
- 失去焦点、按 Escape 或离开输入状态不保存。
- 重复序号使用应用内提示，显示冲突节点，恢复原值，不发送写请求。
- 保存请求只面向桌面端，并携带节点当前版本（`updated_at` 或等价版本标识）；版本已变化时拒绝写入并提示重新加载。
- 历史缺失或重复序号只生成维护告警，不自动重排或修改数据。
- 安卓端本阶段不提供编辑入口，也不参与序号冲突处理；后续同步协议单独定义 `display_order` 语义。

## 数据与接口边界

- 列表读取继续使用现有节点 API，依赖 `updated_at`、`display_order`、Area、Track 和状态字段。
- 需要新增一个桌面端序号更新 API，写入前验证：节点存在、版本匹配、序号为正整数、同 Area + Track 唯一。
- 冲突和版本过期返回结构化错误，前端不将失败误判为成功。
- 写入成功后返回更新后的节点，前端以服务端结果重新排序，不本地猜测其它行的顺序。

## 验收标准

- Library 首屏不再出现连续的大卡片墙；节点行可快速扫描。
- 日期分组边界符合北京时间自然日定义，分组互斥且空组隐藏。
- 最新内容默认位于顶部；刷新、搜索、筛选后顺序稳定。
- “全部展开/收起”只影响当前可见分组。
- 搜索不遮挡内容，URL 可恢复搜索和排序状态。
- 序号编辑只有 Enter 提交；重复序号和版本冲突均不写入并有明确提示。
- 桌面端回归覆盖分组、排序、搜索 URL、折叠状态和序号冲突；安卓端行为保持不变。
