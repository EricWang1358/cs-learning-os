# CS Learning OS 使用说明书

## 日常启动

在项目根目录运行：

```powershell
.\scripts\dev.ps1
```

这个脚本会：

- 停掉旧的 `8000` / `5173` 服务
- 重建 SQLite 索引
- 启动 FastAPI
- 启动 React/Vite
- 打开浏览器

只想关闭服务时：

```powershell
.\scripts\stop-dev.ps1
```

## 阅读知识点

左侧是主入口：

- `All nodes`：全部知识点
- `CS fundamentals`：计算机基础
- `Practice / Quiz Bank`：题库
- `Q Queue`：待读者疑问队列

进入 `CS fundamentals` 后，会出现二级 reading tracks，例如：

- `C and memory`
- `GDB debugging`
- `x86-64 assembly`
- `Bomb Lab`

阅读顺序由 Markdown frontmatter 的 `track` 和 `order` 决定，不写死在前端。

## Focus Reading

点击详情页右上角 `Focus reading` 后，会进入专注阅读模式。

专注模式下：

- 左侧地图和中间列表会收起
- 正文区域变宽
- 右侧出现 `Q to be solved` 疑问框

## Q to be solved

阅读时，如果某段没讲清楚，直接写进右侧 `Q to be solved`。

保存后：

- 问题进入 SQLite 的 `reader_questions` 表
- 当前 node 或 quiz 会显示 `Q to be solved: N`
- `Q Queue` 会集中显示所有 open questions

这能避免之后 AI 遍历所有节点和 tag，直接从队列处理未解决问题。

## Edit Mode

点击详情页右上角 `Edit mode` 可以直接编辑当前 Markdown 正文。

当前版本只编辑 body，不编辑 frontmatter。

适合在浏览器里改：

- 教程正文
- 例子
- 解释
- 常见错误
- 题解步骤

不建议在浏览器里改：

- `title`
- `track`
- `order`
- `tags`
- `linked_nodes`
- `sources`

这些 metadata 仍建议直接改 Markdown 文件。

## Quiz Bank

具体题目放在：

```text
<active-content-root>/quizzes/<area>/
```

题目使用 `Standard Q`。

每道题应该包含：

- Prompt
- Answer
- Explanation
- Plain Explanation
- What This Tests
- Linked Review

题解不要跳步，要写出逐行状态变化、思路翻译、分支判断和必要的十六进制运算。

## 给 AI 的操作说明

处理新增内容前：

1. 先判断是知识点还是题目。
2. 知识点用 `Standard A`。
3. 具体练习题、考试截图、每日复习候选用 `Standard Q`。
4. 如果用户没有指定标准，应询问使用哪个标准。

处理 `Q Queue` 时：

1. 直接读取 open reader questions，不要扫描所有 node/tag。
2. 对每个问题判断是更新原节点，还是新增可复用知识节点。
3. 修改 Markdown 后，重新 ingest SQLite。
4. 如果问题已被解决，应调用 resolve API 或更新其状态。

处理浏览器 edit mode 时：

1. 只保存 Markdown body。
2. 不要通过浏览器编辑 frontmatter。
3. 保存前应确认用户意图。

提交前：

1. 检查 `git status --short`。
2. 不提交 `.venv/`、`node_modules/`、`var/`、`generated/`。
3. 至少运行：

```powershell
backend\smoke_test.py
npm run lint
npm run build
```
