# CS Learning OS 使用说明书

## 朋友测试版启动

第一次使用或环境损坏时，双击项目根目录的 `安装或修复 Learning OS Beta.cmd`。

日常启动时，双击项目根目录的 `启动 Learning OS Beta.cmd`。

`启动 Learning OS Beta.cmd` 会先做依赖自检。如果发现 `.venv`、后端 Python 包或前端 `node_modules` 缺失，会自动调用 bootstrap 修复后再启动。

这个入口适合发给朋友试用：

- 默认使用 `%USERPROFILE%\CSLearningOS` 作为个人数据目录。
- 首次启动会把 `content-demo/` 复制到 `%USERPROFILE%\CSLearningOS\content`。
- SQLite 数据库写入 `%USERPROFILE%\CSLearningOS\knowledge.db`。
- 导出文件写入 `%USERPROFILE%\CSLearningOS\exports`。
- 运行缓存写入 `%USERPROFILE%\CSLearningOS\generated`。
- AI 功能默认关闭，不会使用开发者的 Codex 或 OpenAI API。

关闭服务：

```powershell
.\scripts\stop-dev.ps1
```

只做自检、不安装、不启动：

```powershell
.\scripts\verify-beta.ps1
```

## 开发启动

在项目根目录运行：

```powershell
.\scripts\dev.ps1
```

这个脚本会停止旧的 `8000` / `5173` 服务，重建 SQLite 索引，启动 FastAPI 和 React/Vite，并打开浏览器。

开发模式会优先使用 `CS_LEARNING_DATA_ROOT`，否则使用项目内的 `data/`。如果你想明确指定目录：

```powershell
.\scripts\dev.ps1 -ContentDir D:\path\to\content -DbPath D:\path\to\knowledge.db
```

## 数据目录

学习内容采用 Anki-like package 思路：Markdown 是可读、可迁移的内容包，SQLite 是本地运行索引和学习状态。

常用目录：

- `content-demo/`：演示内容，可随代码分发。
- `data/content/`：开发者本机私有内容。
- `%USERPROFILE%\CSLearningOS\content`：朋友测试版默认内容目录。
- `knowledge.db`：搜索索引、阅读状态、问题队列、复习状态等本地运行数据。
- `assets/`：Markdown 辅助图片和媒体文件。

不要把 `.venv/`、`app/node_modules/`、`generated/`、`var/`、私有 `data/` 当作分发内容。

## 阅读知识点

左侧是主导航：

- `All nodes`：全部知识点。
- `Practice / Quiz Bank`：题库。
- `Daily review`：每日复习。
- `Q Queue`：阅读时保存的问题队列。
- `Knowledge navigator`：知识图谱导航。
- `System health`：本地健康检查和导出入口。

进入详情后，点击 `Focus reading` 可以进入专注阅读模式。专注模式会收起地图和列表，让正文更宽，并显示 `Q to be solved`。

## Q to be solved

阅读时遇到没讲清楚的地方，写进右侧 `Q to be solved` 并保存。

保存后：

- 问题进入 SQLite 的 `reader_questions`。
- 当前 node 或 quiz 会显示 open question 数量。
- `Q Queue` 会集中显示所有未解决问题。

朋友测试版里 AI 默认关闭，所以 `Draft with AI` 会显示为 disabled。问题仍然会正常保存，适合后续人工整理或在可信本机开启 AI 后处理。

## Edit Mode

点击详情页右上角 `Edit mode` 可以直接编辑当前 Markdown body。

稳定性规则：

- 只编辑正文 body，不在浏览器里编辑 frontmatter。
- 保存前不要离开页面；如果离开，系统会提示是否丢弃未保存内容。
- 未保存草稿会按目标和 body hash 临时保存在浏览器本地，用于刷新后恢复。
- 保存后后端通过 ContentWriteService 写 Markdown 和 SQLite，避免双写绕路。

适合在浏览器里改：

- 教程正文
- 例子
- 解释
- 常见错误
- 题解步骤

建议直接改 Markdown 文件：

- `title`
- `track`
- `order`
- `tags`
- `linked_nodes`
- `sources`

## Quiz Bank 和 Daily Review

题目放在：

```text
<active-content-root>/quizzes/<area>/
```

题目使用 `Standard Q`，应包含 Prompt、Answer、Explanation、Plain Explanation、What This Tests 和 Linked Review。

`Daily review` 使用 Anki-like scheduler。当前目标是够用、可解释、轻量，不引入复杂 FSRS 或 embedding 核心依赖。

## System Health 和导出

`System health` 用于检查：

- 内容数量和 SQLite 大小
- generated/export 大小
- schema 版本
- repair report
- AI preflight 状态
- package export
- LLM Wiki pack export

朋友测试版中，导出和缓存默认写入 `%USERPROFILE%\CSLearningOS`，不会污染项目仓库。

## AI 设置原则

朋友测试版默认：

```powershell
CS_LEARNING_AI_ENABLED=false
```

这会阻止：

- `POST /api/ai/revise`
- `POST /api/ai/jobs`
- `POST /api/ai/jobs/{id}/retry`
- Codex/OpenAI preflight credential 准备

已有的本地 AI job 记录仍可查看、取消、拒绝或应用，因为这些操作不再调用外部 provider。

只有在可信本机上，才手动开启：

```powershell
$env:CS_LEARNING_AI_ENABLED="true"
$env:CS_LEARNING_AI_PROVIDER="openai-api"
$env:OPENAI_API_KEY="sk-..."
```
